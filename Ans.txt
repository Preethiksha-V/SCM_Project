# 🔹 1. Core System Design (Java)

### 📌 Key Concepts

* Product → has stock
* Order → processes purchase
* Inventory → updates stock
* Payment → success/failure simulation

---

## ✅ Product Class

```java
public class Product {
    int id;
    String name;
    int stock;
    double price;

    public Product(int id, String name, int stock, double price) {
        this.id = id;
        this.name = name;
        this.stock = stock;
        this.price = price;
    }
}
```

---

## ✅ Inventory Management

```java
import java.util.ArrayList;

public class Inventory {
    ArrayList<Product> products = new ArrayList<>();

    public void addProduct(Product p) {
        products.add(p);
    }

    public Product getProduct(int id) {
        for (Product p : products) {
            if (p.id == id) {
                return p;
            }
        }
        return null;
    }

    public boolean reduceStock(int id, int qty) {
        Product p = getProduct(id);
        if (p != null && p.stock >= qty) {
            p.stock = p.stock - qty;
            return true;
        }
        return false;
    }
}
```

## ✅ Payment Service

public class PaymentService {
    public boolean processPayment(double amount) {
        // simple logic
        if (amount > 0) {
            return true;

---

## ✅ Order Service

```java
public class OrderService {
    Inventory inventory;
    PaymentService payment;

    public OrderService(Inventory inventory, PaymentService payment) {
        this.inventory = inventory;
        this.payment = payment;
    }

    public String placeOrder(int productId, int qty) {
        Product p = inventory.getProduct(productId);

        if (p == null) {
            return "Product not found";
        }

        if (!inventory.reduceStock(productId, qty)) {
            return "Insufficient stock";
        }

        double total = p.price * qty;

        if (!payment.processPayment(total)) {
            return "Payment failed";
        }

        return "Order successful";
    }
}
```

## ✅ Test Class

```java
import org.junit.jupiter.api.Test;
import static org.junit.jupiter.api.Assertions.*;

public class OrderServiceTest {

    @Test
    public void testOrderSuccess() {
        Inventory inv = new Inventory();
        inv.addProduct(new Product(1, "Phone", 10, 1000));

        OrderService os = new OrderService(inv, new PaymentService());

        String result = os.placeOrder(1, 2);

        assertEquals("Order successful", result);
    }

    @Test
    public void testInsufficientStock() {
        Inventory inv = new Inventory();
        inv.addProduct(new Product(1, "Phone", 1, 1000));

        OrderService os = new OrderService(inv, new PaymentService());

        String result = os.placeOrder(1, 5);

        assertEquals("Insufficient stock", result);
    }

    @Test
    public void testProductNotFound() {
        Inventory inv = new Inventory();

        OrderService os = new OrderService(inv, new PaymentService());

        String result = os.placeOrder(99, 1);

        assertEquals("Product not found", result);
    }
}
```

# 🔹 3. CI/CD Pipeline

---

## ✅ GitHub Setup

1. Create repo on GitHub
2. Push project:

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin <repo-url>
git push -u origin main
```

---

## ✅ GitHub Actions (CI)

📄 `.github/workflows/ci.yml`

```yaml
name: Java CI

on:
  push:
    branches: [ "main" ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout
      uses: actions/checkout@v3

    - name: Set up Java
      uses: actions/setup-java@v3
      with:
        distribution: 'temurin'
        java-version: '17'

    - name: Build with Maven
      run: mvn clean install
```
